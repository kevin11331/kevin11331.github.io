Deployment.yaml

Docker指令：

in /Users/liukaixiang/my_stock_price

  1. docker build -t my-stock-price
  2. docker login
  3. docker tag my-stock-price kevin11331/my-stock-price:1.0

上傳到docker hub

  • docker push kevin11331/my-stock-price:1.0

在本地端運行這支container

  • docker run -p 5000:5000 my-stock-price http://localhost:5000

K8s指令：

  1. kubectl apply -f stock_deployment.yaml
  2. kubectl get pods
  3. kubectl get pods -o wide
  4. kubectl get svc
  5. kubectl describe pod my-stock-price-XXX
  6. kubectl describe svc my-stock-price-service
  7. kubectl logs [pod-name]

檢查minikube狀態指令：

  • minikube ip
  • minikube status
  • minikube tunnel
  • minikube dashboard
  • minikube logs
  • minikube service my-stock-price-service
  • minikube service my-stock-price-service –url
kubectl exec-it my-stock-price-588574cf8d-7tp8b -- /bin/sh 
curl http://127.0.0.1:5000

在minikube內部連線到pod上

kubectl port-forward svc/my-stock-price-service 5001:5000
打開瀏覽器：http:localhot:5001

其他檢查指令

minikube ssh sudo ip a

kubectl get events

檢查Flask應用是否監聽port 5000 kubectl exec -it – netstat -tuln | grep 5000

確認Pod是否正確標籤並分配到Service上 kubectl get pods –show-labels
